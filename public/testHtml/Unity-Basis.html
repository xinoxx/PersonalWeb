File 文件菜单

Build Settings：发布项目

当选择Platform中的Pc,Mac...选项时，点击Player Setting，可以修改发布的一些属性。

  • 点击Resolution and Presentation，当中可以修改是否是全屏模式、是否使用本地的分辨率，是否在后台运行等等。为了方便测试，建议Run In Background开启。

  • 点击Splash Image，更改过场图片。
  • 点击Other Setting，更改游戏渲染方式。Rendering Path渲染模式，有Vertex Lit顶点渲染，Forward正项渲染，Deferred Lighting延迟渲染。他们的效果，消耗的配置依次递增。建议使用Forward，因为Vertex Lit不支持阴影等其他效果。Deferred Lighting不支持移动等，且Layers只有4层。

渲染管线

计算机图形学：渲染管线：图形数据在GPU上经过运算处理，最后输出到屏幕的过程。

CPU：（中央处理器，做运算）游戏->图形API

xxxxxxxxxx
CPU判断当前游戏状态下哪一个图形需要进行渲染，交给图形API，API交给GPU。


绘制调用Draw Call，每次引擎准备数据并通知GPU的过程。通俗讲，每帧调用显卡渲染物体的次数。Draw Call越大，说明渲染次数越多，越消耗性能。


点击Unity的Game窗口的Stats，Batches数量就是就是Draw Call的次数。（不考虑光照的情况下，一个物体，一个Draw Call。）Tris:三角的面数，Verts定点数（坐标）。

GPU：（做渲染）顶点处理->图元装配->光栅化->像素处理->缓存

xxxxxxxxxx
顶点处理：
1.接受模型的顶点数据。（接受CPU传递的数据）
2.坐标系转换。
图元装配：
1.组装面：连接相邻的顶点，绘制为三角面。
光栅化：
1.计算三角面上的像素，并为后面着色阶段提供合理的插值参数。
像素处理：
1.对每个像素区域进行着色；
2.写入缓存中。
缓存：
1.一个存储像素数据的内存块，最重要的缓存是帧缓存与深度缓存。
2.帧缓存：存储每个像素的色彩，即渲染后的图像。帧缓存常常存在显存中，显卡不断读取并输出到屏幕中。
3.深度缓存 z-buffer：存储像素的深度信息，即物体到摄像机的距离。光栅化时便计算各像素的深度值，如果新的深度值比现有值更近，则像素颜色被写到帧缓存，并替换深度缓存。

Occlusion Culling

即时遮挡剔除 Instant Occlusion Culling

遮挡剔除：当物体被送进渲染流水线之前，将摄像机视角内看不到的物体进行剔除，从而减少了每帧渲染数据量，提高渲染性能。

（消耗CPU性能，一般用在存在密集物体的情况。）

LOD

Levels of Deatail 多细节层次

LOD技术指根据物体模型的节点在现实环境中所处的位置和重要度，决定物体渲染的资源分配，降低非重要物体的面数和细节度，从而获得高效率的渲染运算。

（缺点：距离判断也是消耗CPU性能，物体变多（迫害美工），占内存）

（存在距离远近变化，LOD使用才有意义，空间换时间）

容量的单位

  • 位bit(比特)：电脑记忆体中的最小单位，每一位由0、1表示
  • 字节Byte：电脑中存储的最小单位（1Byte=8bit，1KB(千字节)=1024Byte，1MB=1024KB，G、T）
  • 网速10M指的是Mbps(兆位/秒)是速率单位，换算成字节应该是10/8=1.25MBps兆字节/秒

类型分类

CTS(Common Type System)：

  • 值类型：存储数据本身
  • 引用类型：存储数据的引用（内存地址）

基本类型

整型（整数）

  • 1个字节：有符号sbyte(-128-127)，无符号byte(0-255)
  • 2个字节：有符号short(-32768-32767)，无符号ushort(0-65535)
  • 4个字节：有符号int，无符号uint
  • 8个字节：有符号long，无符号ulong

非整数（小数）

  • 4字节：单精度浮点 float 精度7位

  • 8字节：双精度浮点 double 精度15-16位

  • 16字节：128位数据类型decimal 精度28-29位，适用于财务和货币运算。

  • 注意事项：1.非整型变量赋值要加入后缀（1.2f 1.2d 1.2m)，不加默认double

    2.浮点型运算会出现舍入误差： bool number = 1.0f-0.9f == 0.1f；number是false

    二进制无法精确表示1/10，就像十进制无法精确表示1/3，所以二进制表示10进制会有一些舍入误差，对于精度要求较高的场合会导致代码的缺陷，可以使用decimal代替。

非数值型：

  • char 字符，2字节
  • string 字符串
  • bool 类型，1字节

        Console.WriteLine(o2); // o2为1
        
        // 替换引用，方法外不会受到影响，应该是通过引用修改堆中的数组对象
        // 在方法内修改堆中的数组对象，可以使用CopyTo
        newArray.CopyTo(array, 0);
        // 内部代码就是array[0] = newArray[0]的循环赋值。
    }
    
    // params 参数数组
    // 对于方法内部而言：就是个普通数组
    // 对于方法外部（调用者）而言：可以传递数组、传递一组数据类型相同的变量集合
    // 甚至可以不传递参数
    // 作用：简化调用者调用方法的代码
    private int Add(Params int[] arr)
    {
        int sum = 0 ;
        foreach(int item in arr)
        {
            sum += item;
        }
        return sum;
    }
    
    // 值参数：按值传递 -- 传递实参变量存储的内容
    // 作用：传递信息
    private static void Fun1(int a){}
    
    // 引用参数：按引用传递 -- 传递实参变量自身的内存地址
    // 作用：改变数据
    // 调用该方法时，int a = 1; Fun2(a); 会报错
    private static void Fun2(ref int a)
    {
        // 方法内部修改引用参数，实质上就是修改实参变量
        a = 2;
    }
    // 正确调用方式：Fun2(ref a); 调用之后结果a = 2;
    
    // 输出参数：按引用传递 -- 传递实参变量自身的内存地址
    // 作用：返回结果（可以返回多个结果，在方法参数处可以定义多个out参数)
    // 可以直接定义参数，不赋值，用于接受输出参数结果。
    private static void Fun3(out int a)
    {
        a = 2;
    }
    
    // ref与out的区别：
    // 1.方法内部必须为输出参数赋值
    // 2.输出参数传递之前可以不赋值
    
    int result;
    // TryParse方法，out:转换后的结果  返回值：是否可以转换
    bool re = int.TryParse("250+", out result); // result:0
}

    builder.Insert(0,"builder");

抽象类和接口有什么区别：

继承接口的类必须实现这个类的所有方法，相当于规则，且接口不能有成员变量或字段。

抽象类里不止存在抽象方法，还有非抽象方法，并且不用重写抽象类的方法，抽象类不能被实例化。

相当于是一种规范。
