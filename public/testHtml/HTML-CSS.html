CSS浮动

浮动的特点

1、浮动元素会脱离标准流，在标准流中不占用位置。

2、这里添加浮动属性后，浮动元素比标准流高半个级别，可以覆盖标准流中的元素，但不会覆盖其文字（因为有浮动，margin: 0 auto不会生效）。

3、浮动找浮动， 下一个浮动元素会在上一个浮动元素后面左右浮动。

4、浮动元素特殊显示效果：一行可以显示多个；可以设置宽高。

5、浮动：在一行排列，宽高生效，浮动后的标签具备行内块特点，如果父级的宽度不够，子级会自动换行。

6、浮动的标签，默认顶对齐。

清除浮动影响（子元素浮动脱标）

Q：子级浮动，父级没有高度，后面的标准流会受影响，显示到上面的位置。

A：1、给父级加高度

2、如果高度不固定，额外标签法：在父元素里最后添加一块元素，给添加的块级元素设置：clear:both

3、单伪元素清除法（常用）

4、双伪元素清除法

5、给父元素设置overflow:hidden

xxxxxxxxxx
/* 双伪元素清除法 */
/* .clearfix::before 作用:解决外边距塌陷问题
外边距塌陷:父子标签,都是块级,子级加margin会影响父级的标签
*/
.clearfix::before,
.clearfix::after {
    content: '';
    display: table;
}
/* 真正执行清除浮动标签 */
.clearfix::after {
    clear: both;
}

CSS三大特性：

1、层叠性：

  • 后面的内容覆盖前面的内容。
  • 当选择器优先级相同时，才通过层叠性判断。

2、继承性：

控制文字的属性都能够继承。

a标签本身有颜色，则不会继承颜色属性。

h系列标签不会继承font-size属性。

3、优先级：

（范围越广，优先级越低）

继承 < 通配符 < 标签 < 类选择器 < id选择器 < 行内样式 < !important

CSS定位

1、网页布局：标准流、浮动、定位

不同布局方式元素的层级关系：标准流 < 浮动 < 定位

不同定位之间的层级关系：相对、绝对，固定默认层级相同，此时写在下面的元素层级更高，除此之外可以使用z-index来设置层级，数字越大层级越高。默认的z-index为0，且z-index必须和position配合使用。

xxxxxxxxxx
.one {
    position: absolute;
    z-index:1;
}

2、定位方式：

position：static静态定位\relative相对定位\absolute绝对定位\fixed固定定位。

一般父级相对定位，子级绝对定位，子绝父相。

相对位置relative

  1. 占有原来的位置(没有脱标)。
  2. 仍然具有标签原有的显示模式特点。
  3. 改变位置参照自己原来的位置。
  4. 如果left/right都有，以left为准，同理，以top为准。
  5. 如果不写left等方位词，就类似标准流，不会有任何影响。

绝对定位absolute

先找已经定位的父级，如果有这样的父级就以这个父级为参照物进行定位；如果有父级，但父级没有定位，或者没有父级，则以浏览器窗口为参照物定位。

  1. 不占位置，脱标。如果没有父级，且没有写left等方位词，会脱标。
  2. 改变标签的显示模式特点：具备行内块的特点(在一行共存，宽高生效)。
  3. 查找父级的方式，就近找定位的父亲，如果逐层都找不到定位了的父级，则就以浏览器来定位。

固定定位fixed

相对于浏览器定位 ，脱标，改变位置参考浏览器窗口，具备行内块特点,尽量设置宽高。

3、定位-居中

Q：绝对定位的盒子，不能使用margin: 0 auto来居中了。

A：法1：可以通过margin按照百分比配合position使用。

法2：通过transform来修改。

xxxxxxxxxx
/* 实现位移,位移自身的一半 */
transform: translate(-50%, -50%);

4、装饰

处理行内块和文字/行内块对齐，都是用vertical-align:middle控制(是因为浏览器默认是以基线对齐) 。或者将行内块元素修改为块元素，display:block。垂直对齐: vertical-align。

浏览器处理行内块元素都当作文字处理，因此，处理行内块元素的水平位置的居中，可以使用text-align:center来控制(比如img等)。

Attention & Question!!!

  • CSS书写顺序：浮动 display 盒子model(margin border padding 宽度高度背景) 文字样式

  • CSS代码中，上一行代码出错，也会导致下一行代码出错。

  • 精灵图，一般使用行内标签，使用background-position来协助改变图片位置，不适用img。

  • 系统Q：浏览器解析行内块或行内标签时，如果标签换行书写，会产生一个空格的间距。

    A：设置浮动。

  • 外边距Q：垂直布局的块级元素，上下的margin会合并，最终两者的距离取决于两者margin值的最大值。

    A：只给一个盒子设置margin值。

  • 外边距Q：塌陷现象：互相嵌套的块级元素，子元素的margin-top会作用在父元素上，导致父元素一起移动。

    A：给父级设置overflow:hidden

    转换成行内元素、设置浮动、给父级添加padding-top值

  • 行内元素：margin和padding改变垂直方向的属性无法生效，若要修改垂直方向，可以使用line-height来控制。

  • 优先级权重叠加计算：行内样式个数/id选择器的个数/类选择器的个数/标签选择器的个数；

    1、先比较第一级，如果比较出来不看后后面比较的结果；

    2、如果优先级相同，则比较层叠性，写在下面的决定

    3、如果!important不是继承，则优先级最高

    4、如果都是继承，则看其第一个继承的父级，继承·该父级的样式
